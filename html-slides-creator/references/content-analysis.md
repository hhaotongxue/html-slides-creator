# 内容分析与布局决策

本文档提供清晰的决策树，帮助根据内容特征选择最合适的布局模式。

## 决策流程

```
内容分析 → 确定内容类型 → 评估信息密度 → 选择布局模式 → 应用视觉风格
```

## 第一步：内容类型识别

### 1. 文字为主的内容

| 特征 | 内容类型 | 推荐布局 |
|------|---------|---------|
| 1-2 句话，字数少于 30 字 | 金句/口号 | 居中大字号 + 留白 |
| 引用他人观点，有出处 | 引用 | 引用框布局 |
| 3-6 个并列要点，每个 1-2 行 | 要点列表 | 卡片网格 / 图标列表 |
| 7+ 个要点或长列表 | 详细列表 | 分栏列表 / 多页拆分 |
| 一个主题 + 3-5 个子点 | 总分结构 | 大标题 + 分区块 |
| 问题 + 答案 | Q&A | 对话气泡布局 |

### 2. 对比与比较

| 特征 | 内容类型 | 推荐布局 |
|------|---------|---------|
| 两个方案/概念对比 | 二元对比 | 左右分栏 / VS 布局 |
| 3-4 个选项比较 | 多选项比较 | 对比表格 / 卡片网格 |
| 优缺点分析 | 利弊分析 | 双色分栏（绿/红） |
| 前后变化 | 对比展示 | 时间轴对比 |

### 3. 流程与步骤

| 特征 | 内容类型 | 推荐布局 |
|------|---------|---------|
| 3-5 个线性步骤 | 简单流程 | 横向时间线 |
| 6+ 个步骤 | 复杂流程 | 纵向步骤条 / 多页拆分 |
| 有分支的流程 | 决策树 | 流程图布局 |
| 循环流程 | 循环过程 | 环形布局 |

### 4. 数据与数字

| 特征 | 内容类型 | 推荐布局 |
|------|---------|---------|
| 1-3 个关键数字 | 数据亮点 | 大数字展示 |
| 4-6 个指标 | 数据面板 | 指标卡片网格 |
| 数据对比 | 数据比较 | 柱状图风格布局 |
| 百分比/进度 | 进度展示 | 进度条布局 |

### 5. 代码与技术

| 特征 | 内容类型 | 推荐布局 |
|------|---------|---------|
| 代码片段 + 说明 | 代码示例 | 代码块 + 注释 |
| 多个代码片段对比 | 代码对比 | 左右代码对比 |
| 配置文件 | 配置展示 | 代码块 + 参数说明 |
| 命令行操作 | 终端演示 | 终端风格布局 |

### 6. 视觉与图形

| 特征 | 内容类型 | 推荐布局 |
|------|---------|---------|
| 图片 + 说明文字 | 图文混排 | 图片主导布局 |
| 多张图片展示 | 图片画廊 | 网格画廊 |
| 图标 + 文字说明 | 图标说明 | 图标网格 |
| 架构图/示意图 | 图表展示 | 全屏图表 |

## 第二步：信息密度评估

### 密度等级

| 等级 | 特征 | 处理策略 |
|------|------|---------|
| **极简** | 少于 20 字 | 大字号 + 大量留白 + 居中 |
| **稀疏** | 20-100 字 | 中等字号 + 适度留白 |
| **适中** | 100-200 字 | 标准排版 + 分段 |
| **密集** | 200-300 字 | 分栏 / 缩小字号 / 考虑拆页 |
| **过载** | 300+ 字 | **必须拆成多页** |

### 拆页原则

- 一页只讲一个核心观点
- 宁可多一页，不要挤在一起
- 每页信息密度保持在"适中"以下
- 使用章节封面页分隔不同主题

## 第三步：视觉层级设计

### 信息重要性分级

| 级别 | 视觉处理 | CSS 实现 |
|------|---------|---------|
| **核心信息** | 最大字号 + 强调色 + 居中 | `font-size: 3-5rem; color: accent` |
| **主要信息** | 大字号 + 粗体 | `font-size: 2-3rem; font-weight: 700` |
| **次要信息** | 标准字号 + 正常色 | `font-size: 1-1.5rem; color: main` |
| **补充信息** | 小字号 + 灰色 | `font-size: 0.875rem; color: muted` |

### 视觉对比技巧

1. **大小对比**：核心信息字号是次要信息的 2-3 倍
2. **颜色对比**：重点用强调色，普通用灰色
3. **位置对比**：重要内容放上方或中心
4. **留白对比**：重要内容周围留更多空白

## 第四步：情感与氛围

### 情感色彩匹配

| 情感 | 视觉策略 | 适用风格 |
|------|---------|---------|
| **兴奋/活力** | 明亮色彩 + 动态元素 | Dopamine |
| **专业/严肃** | 深色 + 简洁排版 | Jelly |
| **温暖/友好** | 柔和色彩 + 圆润形状 | Macaron |
| **紧急/警告** | 红色/橙色 + 醒目标识 | 所有风格 + 警告色 |
| **成功/完成** | 绿色 + 勾选图标 | 所有风格 + 成功色 |

## 决策树示例

### 示例 1：用户输入

```
内容：Docker 的三大核心概念：镜像、容器、仓库
```

**决策过程**：
1. 内容类型：3 个并列要点 → 要点列表
2. 信息密度：约 15 字 → 极简
3. 布局选择：卡片网格（3 列）
4. 视觉处理：大图标 + 大字号 + 简短说明

### 示例 2：用户输入

```
内容：传统部署 vs 容器化部署的对比
传统：环境配置复杂、部署慢、难以迁移
容器：一键部署、秒级启动、跨平台
```

**决策过程**：
1. 内容类型：二元对比 → 对比类
2. 信息密度：约 50 字 → 稀疏
3. 布局选择：左右分栏 + VS 标识
4. 视觉处理：左侧灰色（旧）+ 右侧强调色（新）

### 示例 3：用户输入

```
内容：Docker 安装步骤（共 8 步，每步有详细说明）
```

**决策过程**：
1. 内容类型：复杂流程（8 步）
2. 信息密度：预计 200+ 字 → 密集
3. 布局选择：拆成 2-3 页，每页 3-4 步
4. 视觉处理：纵向步骤条 + 数字标识

## 特殊场景处理

### 封面页

- 必须包含：主标题 + 副标题（可选）
- 视觉策略：超大字号 + 渐变文字 + 装饰元素
- 留白比例：至少 50%

### 章节分隔页

- 用于分隔不同主题
- 视觉策略：中等字号 + 居中 + 简洁
- 可以只有一个大标题

### 结束页

- 必须包含：总结语 / 感谢语 / 联系方式
- 视觉策略：温和色彩 + 居中 + 适度装饰

## 使用建议

1. **先分析，后选择**：不要直接套用模板，先理解内容特征
2. **保持一致性**：同一类型内容在整个幻灯片中使用相同布局
3. **节奏变化**：避免连续多页使用相同布局，适当穿插不同密度的页面
4. **测试可读性**：生成后检查文字是否清晰、信息是否易懂
